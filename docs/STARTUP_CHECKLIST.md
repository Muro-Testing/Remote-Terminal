# Startup Checklist

## First chat
- Confirm project goal and success criteria.
- Confirm V1 scope and out-of-scope.
- Confirm users/roles and permissions model.
- Confirm stack and integrations.
- Confirm timeline and budget style.

## Before coding
- Ensure `PROJECT_CONTEXT.md` is filled and confirmed.
- Create first plan file in `docs/` with dated checklist.
- Lock first implementation task.

## During execution
- One task at a time.
- Confirm continuation after each completed task when task list is long.
- Keep context updated when major decisions change.

## Before push
- Summarize changes.
- Confirm include/exclude files.

## After push
- Update changelog entry.
- Keep changelog local if workflow requires.
